# 告警中心

## 概述

告警引擎每分钟评估规则，满足条件时发送通知。

## 告警规则

### 高错误率告警

Provider 错误率超过阈值时触发：

| 字段 | 说明 |
|------|------|
| 阈值 | 错误率（如 0.1 = 10%）|
| 窗口 | 评估时间窗口（分钟）|

### 额度预警

Token 剩余额度低于阈值时触发：

| 字段 | 说明 |
|------|------|
| 阈值 | 剩余百分比（如 0.2 = 20%）|
| 范围 | token 或 organization |

### Provider 断开告警

Provider 持续失败时触发：

| 字段 | 说明 |
|------|------|
| 最大重试 | 连续失败次数 |
| 窗口 | 评估时间窗口（分钟）|

## 告警级别

| 级别 | 颜色 | 行为 |
|------|------|------|
| `critical` | 红色 | 全部渠道通知 |
| `warning` | 黄色 | 通知，不升级 |
| `info` | 蓝色 | 仅通知 |

## 通知渠道

### 邮件

在 **系统设置** → **告警设置** 中配置 SMTP。

### 微信

使用 ServerChan（方糖推送）实现微信通知。

## 冷却

默认冷却时间 5 分钟，避免告警轰炸。
