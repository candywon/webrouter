# 隐私脱敏

## 概述

WebRouter 的脱敏引擎在请求到达上游 Provider 前，自动检测并替换个人身份信息（PII），响应中再还原原始数据。上游永远看不到敏感信息。

## 内置规则

| 规则 | 分类 | 检测方式 |
|------|------|----------|
| 手机号 | `PHONE` | 正则匹配 |
| 姓名 | `NAME` | 中文姓名正则 |
| 地址 | `ADDRESS` | 地址关键词正则 |
| 身份证 | `IDCARD` | 模式匹配 |
| 邮箱 | `EMAIL` | 正则 |

## 工作原理

1. **转发前** — wr-proxy 扫描请求体中的 PII 模式
2. **替换** — 匹配内容替换为占位符（`[PHONE_1]`、`[NAME_2]` 等）
3. **转发** — 脱敏后的请求发送到上游
4. **响应** — 上游正常返回
5. **还原** — wr-proxy 将占位符恢复为原始值，返回给客户端

## 自定义规则

| 字段 | 说明 |
|------|------|
| 名称 | 规则标识 |
| 类型 | `regex` |
| 正则 | 匹配模式 |
| 分类 | PHONE、NAME、ADDRESS、EMAIL、CUSTOM |
| 级别 | `standard` 或 `strict` |

## 配置

脱敏按 Token 启用。创建或编辑 Token 时，打开**脱敏开关**即可为该 Token 开启请求脱敏。
