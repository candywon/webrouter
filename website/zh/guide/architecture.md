# 架构概览

## 组件架构

WebRouter 由两个主要组件构成：

```
┌─────────────┐     HTTP      ┌─────────────────┐
│  浏览器/CLI  │ ───────────→ │    WebRouter     │
│             │ ←──────────── │    (Flask)       │
└─────────────┘               │    :5050         │
                              └──────┬──────────┘
                                     │
                              ┌──────▼──────┐
                              │  wr-proxy    │
                              │  (Go) :5051  │
                              └──────┬──────┘
                                     │
                     ┌───────────────┼───────────────┐
                     │               │               │
              ┌──────▼──────┐ ┌──────▼──────┐ ┌──────▼──────┐
              │   direct    │ │  aggregate  │ │   custom    │
              │  (官方直连)  │ │  (聚合平台)  │ │ (自定义网关) │
              └─────────────┘ └─────────────┘ └─────────────┘
```

## 组件说明

### WebRouter（Flask 后端）

管理面板和 REST API 服务：

- **端口：** 5050
- **语言：** Python Flask
- **职责：** 数据源管理、令牌管理、健康监控、计费、团队管理、系统设置
- **数据库：** SQLite（默认）或 MySQL/PostgreSQL
- **调度器：** APScheduler，定时健康检测和告警评估

### wr-proxy（Go 代理网关）

高性能请求代理：

- **端口：** 5051
- **语言：** Go 1.22
- **职责：** 请求转发、智能路由、退避重试、隐私脱敏、流式传输、成本计量

## 数据流

1. **客户端发送请求** 到 wr-proxy（携带 WebRouter API Key）
2. **wr-proxy 认证** Token 并基于模型或 `auto` 路由选择最优 Provider
3. **脱敏引擎** 从请求体中剥离 PII
4. **转发** 请求到上游 Provider
5. **计量** Token 消耗和成本
6. **返回** 响应给客户端（如有需要则还原脱敏数据）
