# wr-proxy 数据流总览

```
客户端请求
    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│ 1. HTTP 入口 (handlers.go)                                  │
│    /v1/chat/completions, /v1/embeddings, /v1/models ...     │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 2. 鉴权 (auth.go)                                           │
│    Bearer Token → LoadTokenByKey → 检查启用/过期/配额/IP白名单 │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 3. 智能模型选择 (smart_model.go)                              │
│    auto/smart别名 → 6维复杂度评分 → 模型分级匹配               │
│    smart_downgrade → 简单请求自动降级到便宜模型                 │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 4. Provider 路由 (router.go)                                 │
│    过滤可用 → 优先级分组(主力/热备/冷备) → 调度策略选Provider   │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 5. 请求校验+清洗 (sanitizer.go)                              │
│    必要字段校验 → messages补全 → tools剥离(按Provider能力)      │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 6. 上游转发 (proxy.go)                                       │
│    Forward() → 构造上游请求 → 发送 → 错误语义分析              │
│    StreamResponse() → SSE逐行透传 → 截断/错误检测             │
│    NonStreamResponse() → 整体转发 → 截断/JSON完整性检测        │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 7. 智能重试引擎 (retry.go)                                    │
│    ShouldFailover → 截断?额度用完?限流?超时? → 决策            │
│    截断 → 同Provider增大max_tokens重试                        │
│    额度用完 → 冷却30min + 换Provider                          │
│    长时限流 → 冷却到预计恢复 + 换Provider                      │
│    短时限流/超时 → 同Provider重试(0.5s递增等待)                 │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 8. 统计计量 (meter.go + cost.go)                             │
│    RecordRequest → 写DB日志 + 扣Token配额 + 累加Provider用量   │
│    CalculateCost → 定价表查价格 → 计算本次请求成本              │
└─────────────┬───────────────────────────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────────────────────────┐
│ 后台常驻                                                      │
│  health.go  → 每5分钟检测Provider健康                         │
│  predictor.go → 额度消耗预测 + 耗尽日期推算                    │
│  alert.go   → 告警评估 + 冷却去重 + 通知                      │
│  reqCache   → 每10分钟清理过期Hash缓存                        │
└─────────────────────────────────────────────────────────────┘
```
