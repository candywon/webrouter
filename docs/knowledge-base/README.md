## 架构总览

```
wr-proxy (Go)
  ├── handleProxy() 现有流程不变
  └── [新增] 异步捕获 → Knowledge Worker → 信号筛选 → 写入raw表

Flask管理后台 (Python)
  ├── LLM异步提炼（qwen3-coder-flash批量处理）
  ├── 数据点提取 + 原文锚定（factual专有）
  ├── 二次机器验证（value在原文中逐字匹配）
  ├── Embedding向量化 + 语义检索
  ├── 审核队列 + 业务域演化
  └── 知识库管理页面

共享存储：SQLite (webrouter.db)
```

## 关键设计决策

1. **数据只锚定不提取**：factual类型value逐字复制，机器可验证
2. **部门由Token确定**：零误差，不需要LLM猜
3. **业务域可演化**：从数据中生长，pending→confirmed/merged
4. **全异步不阻塞**：channel满则丢，代理性能零影响
5. **多级漏斗过滤**：80%噪音在第1级规则筛选中被丢弃
6. **LLM summary仅用于检索**：不作为数据源，数据源只有value和source_quote
7. **MCP双通道**：REST API给人类，MCP Server给AI智能体，共享后端逻辑，权限自动注入
8. **MCP工具只读**：智能体只能检索，不能修改/删除/审核，管理操作走REST API
9. **超长上下文分析**：单域/跨域知识全量灌入大模型做深度分析，模型自动选择(DeepSeek→Gemini→Qwen-Long)，分段汇总策略，分析结果不入知识库
10. **六维增强体系**：System Prompt预注入(零成本静态) + RAG自动注入(常见问题) + MCP工具(深度查询) + 持久记忆(个性化) + 对话摘要(长对话) + 知识微调(远期)
