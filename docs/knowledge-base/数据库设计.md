# 企业知识库 — 数据库设计

> 版本：v1.0 | 日期：2026-05-14

## 1. 原始对话暂存

LLM 处理前的原始对话暂存。处理完成后标记为 done，定期清理。

```sql
CREATE TABLE IF NOT EXISTS wr_knowledge_raw (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    request_id TEXT NOT NULL,
    token_id INTEGER NOT NULL,
    token_name TEXT DEFAULT '',
    model_name TEXT NOT NULL,
    prompt TEXT NOT NULL,              -- 脱敏后的完整prompt
    response TEXT NOT NULL,            -- 脱敏后的完整response
    turn_count INTEGER DEFAULT 1,      -- 对话轮数
    client_ip TEXT DEFAULT '',
    status TEXT DEFAULT 'pending',     -- pending/processing/done/skipped
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    processed_at DATETIME
);

CREATE INDEX idx_kraw_status ON wr_knowledge_raw(status);
CREATE INDEX idx_kraw_token ON wr_knowledge_raw(token_id);
```

## 2. 知识条目

提取后的结构化知识。

```sql
CREATE TABLE IF NOT EXISTS wr_knowledge_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    -- 知识类型
    type TEXT NOT NULL,                -- factual / analytical / procedural

    -- 提取内容
    title TEXT NOT NULL,               -- LLM生成的标题（仅用于展示/检索）
    summary TEXT DEFAULT '',           -- LLM生成的摘要（仅用于检索）
    domain_code TEXT DEFAULT '',       -- 业务域代码
    department TEXT DEFAULT '',        -- 部门（来自Token）

    -- 原文锚定
    source_request_id TEXT NOT NULL,   -- 原始请求ID
    source_turn_index INTEGER DEFAULT 0,
    source_quote TEXT NOT NULL,        -- 原文片段（精确引用，不改动一字）
    source_char_start INTEGER DEFAULT 0,
    source_char_end INTEGER DEFAULT 0,

    -- 数据类专门字段（仅factual类型使用）
    data_points TEXT DEFAULT '',       -- JSON数组，见下方格式

    -- 验证
    confidence REAL DEFAULT 0.0,       -- 0.0-1.0，LLM自评
    verification TEXT DEFAULT 'auto',  -- auto/pending/verified/rejected
    verified_by INTEGER DEFAULT 0,     -- 审核人ID
    verified_at DATETIME,

    -- 元数据
    token_id INTEGER NOT NULL,
    token_name TEXT DEFAULT '',
    model_name TEXT DEFAULT '',
    sensitivity TEXT DEFAULT 'medium', -- low/medium/high/restricted
    retention_until DATETIME,          -- 保留期限
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_kitem_domain ON wr_knowledge_items(domain_code);
CREATE INDEX idx_kitem_dept ON wr_knowledge_items(department);
CREATE INDEX idx_kitem_type ON wr_knowledge_items(type);
CREATE INDEX idx_kitem_sensitivity ON wr_knowledge_items(sensitivity);
CREATE INDEX idx_kitem_verification ON wr_knowledge_items(verification);
CREATE INDEX idx_kitem_created ON wr_knowledge_items(created_at);
```

### data_points JSON 格式

```json
[
  {
    "metric": "毛利率",
    "value": "18.7%",
    "period": "Q3",
    "unit": "百分比",
    "original_sentence": "Q3毛利率同比收窄3.2个百分点至18.7%",
    "certainty": "certain"
  }
]
```

字段说明：
- `metric`: 指标名称
- `value`: 从原文逐字复制的值（禁止改写/换算/推断）
- `period`: 时间范围
- `unit`: 单位
- `original_sentence`: 包含该数据的原文完整句子
- `certainty`: certain（确定）/ uncertain（不确定）

## 4. 向量索引

```sql
CREATE TABLE IF NOT EXISTS wr_knowledge_vectors (
    item_id INTEGER PRIMARY KEY,       -- 关联 wr_knowledge_items.id
    embedding BLOB NOT NULL,           -- 向量，JSON float数组
    model TEXT DEFAULT '',             -- 使用的embedding模型
    dimensions INTEGER DEFAULT 0,      -- 向量维度
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (item_id) REFERENCES wr_knowledge_items(id) ON DELETE CASCADE
);
```

## 5. Token 扩展字段

在 wr_tokens 表增加知识库捕获开关：

```sql
ALTER TABLE wr_tokens ADD COLUMN knowledge_capture_enabled INTEGER DEFAULT 0;
ALTER TABLE wr_tokens ADD COLUMN knowledge_department TEXT DEFAULT '';
```

- `knowledge_capture_enabled`: 是否开启知识捕获（0/1）
- `knowledge_department`: 该Token归属的部门名称（用于知识自动归类）
