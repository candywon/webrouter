# 需求分析文档

## 一、市场背景

### 1.1 行业现状

AI API 中转站赛道在 2024-2025 年爆发式增长，核心驱动因素：

- 国内开发者调用海外 AI 模型存在网络、支付、账号三重门槛
- One-API (3万+ Star)、New-API (3.2万+ Star)、Sub2API 等开源工具降低了搭建门槛
- 但"能搭起来"和"能稳定运营"之间存在巨大鸿沟

### 1.2 竞品分析

| 产品 | 类型 | 优势 | 短板 |
|------|------|------|------|
| One-API | 开源框架 | 成熟稳定，社区大 | UI 老旧，功能基础，无商业仪表盘 |
| New-API | 开源框架（One-API 增强版） | 新 UI，多语言，缓存计费，支持 Claude/Gemini 原生格式 | 仍是框架级，需用户自己运维 |
| Sub2API | 开源框架（订阅转API） | 解决账号获取问题 | 仅处理账号层，无管理/监控能力 |
| AnyRoute | 商业中转站 | 品牌化，定价透明 | 不卖工具，只卖服务，用户无法自建 |
| cc-switch | 配置工具 | 一键对接 CLI | 仅做配置，无管理/监控/计费 |

**市场空白：** 没有一个产品把"部署 + 管理 + 监控 + 计费 + 对接"全流程串起来。开源项目给你零件，WebRouter 给你整车。

## 二、目标用户

### 2.1 用户画像

**画像 A：独立开发者（占比 50%）**
- 人群：1-3 人小团队，用 Claude Code / Codex 写代码
- 痛点：官方 API 付费门槛高，自己搭中转站但不会运维
- 付费意愿：月付 29-49 元，只要比官方便宜且稳定
- 决策因素：部署简单 > 功能全面 > 价格

**画像 B：自媒体/内容创作者（占比 25%）**
- 人群：用 AI 批量生成内容，日消耗 token 量大
- 痛点：需要多账号轮换防封，但不懂技术，手动管理混乱
- 付费意愿：月付 49-99 元，愿意为省心付费
- 决策因素：省心省事 > 稳定性 > 价格

**画像 C：小团队/企业（占比 15%）**
- 人群：5-20 人技术团队，内部共享 AI API 调用
- 痛点：需要团队额度分配、成本分摊、使用审计
- 付费意愿：月付 99-399 元，需要专业功能
- 决策因素：团队管理能力 > 数据安全 > 稳定性

**画像 D：分销商/二道贩子（占比 10%）**
- 人群：手里有渠道资源，想做中转站卖 API 额度
- 痛点：需要用户管理、充值计费、多渠道调度
- 付费意愿：月付 199-399 元，需要完整的商业化功能
- 决策因素：计费灵活性 > 可扩展性 > 品牌定制

### 2.2 用户规模估算

- 第一阶段（0-3 月）：50-100 用户（免费 + 付费）
- 第二阶段（3-6 月）：200-500 用户
- 第三阶段（6-12 月）：500-2000 用户

## 三、痛点分析

### 3.1 现有开源方案的 6 大痛点

| # | 痛点 | 严重程度 | 现有解决方案 | 方案缺陷 |
|---|------|---------|------------|---------|
| P1 | 部署门槛高 | ★★★★ | Docker 命令 | 对非技术人员仍是墙，出错无从排查 |
| P2 | 多工具对接复杂 | ★★★ | 手动改配置 | Claude Code / Codex / OpenClaw / Hermes 每个配置方式不同 |
| P3 | 运维黑箱 | ★★★★★ | 查日志 | 不知道哪个 key 活着、哪个挂了，被动发现 |
| P4 | 成本不透明 | ★★★★ | 手动算 | 每个模型花了多少钱，没有直观看板 |
| P5 | 团队协作难 | ★★★ | 共享 key | 谁用了多少、该摊多少钱，算不清 |
| P6 | 封号应对迟缓 | ★★★★★ | 人工发现 | key 挂了没有告警，可能断了好几个小时 |

### 3.2 痛点优先级排序

```
P3 运维黑箱 > P6 封号应对 > P1 部署门槛 > P4 成本透明 > P5 团队协作 > P2 多工具对接
```

## 五、非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 仪表盘加载 < 2s，API 代理延迟增加 < 50ms |
| 可靠性 | 监控服务可用性 > 99.5% |
| 安全 | HTTPS 加密，Key 加密存储，用户数据隔离 |
| 可扩展 | 支持水平扩展，模块化设计 |
| 兼容性 | 与 New-API 数据库完全兼容，平滑升级 |
| 部署 | 支持 Docker Compose 一键部署，5 分钟内完成 |

## 六、核心约束

1. **不碰流量：** WebRouter 不中转 API 请求，只提供管理能力，降低合规风险
2. **不碰账号：** 不提供任何上游账号资源，用户自行获取
3. **复用 New-API：** 以 New-API 为 API 网关核心，WebRouter 专注管理/监控/计费层
4. **技术栈一致：** 前端纯 HTML/CSS/JS + Flask 后端，与包装站技术栈统一，降低维护成本
5. **一人可运维：** 初期产品必须做到一个人就能维护运营
