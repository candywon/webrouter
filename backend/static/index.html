💡 智能模型选择说明








✨ model=auto / smart



                    Set the model parameter to auto or smart. The gateway analyzes request complexity across input length, conversation turns, code detection, tool usage, reasoning keywords, and system prompt length, then automatically selects the most suitable model.
                  



Simple Request → Economy model (fast and cost-saving)

Medium Complexity → Standard model (balanced cost/performance)

Complex Reasoning → Advanced model (strongest reasoning)
                  






📝 Usage Example


curl GATEWAY_URL/v1/chat/completions \
  -H "Authorization: Bearer API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"auto","messages":[
    {"role":"user","content":"Analyze the issue in this code"}
  ]}'



                    Smart downgrade can also be enabled per member in Token Management without changing the client-side model parameter.
